# FinalReview-Skill

FinalReview-Skill 是一个给 Codex、Claude Code、WorkBuddy 这类 Agent 使用的期末复习 Skill。

它不是一个普通的 Python 程序，也不是一个“随机帮你出几道题”的提示词合集。它更像是一套期末周专用的复习工作流：你把课程资料交给 Agent，它会按考试得分的逻辑，帮你清洗资料、提炼考点、整理公式、判断重点、生成题目、写答案解析、做模拟卷、安排复习计划，并尽量保留资料来源，方便你回到原文核对。

如果你也有过这种时刻：PPT 一堆、PDF 一堆、作业没整理、往年卷不知道怎么用、老师上课说“这个要考”但你找不到是哪一页，那这个 Skill 就是为这种期末周场景准备的。

## 这个项目解决什么问题

期末复习最痛苦的往往不是“不知道要不要努力”，而是：

* 资料太多，不知道先看哪个。
* PPT、作业、往年卷、课堂转写分散在不同文件里。
* AI 很会讲概念，但不一定知道这门课怎么考。
* 随机生成的题看起来很多，却不像真实考试。
* 答案解析讲得很完整，但不知道考试时到底该写哪几句话。
* 明明资料里有重点，AI 却把所有内容平均对待。

FinalReview-Skill 的设计目标很简单：让 Agent 更像一个会抓重点的期末复习搭子，而不是泛泛而谈的百科助手。

它会优先关注：

1. 真实考试和历年题怎么考。
2. 老师 PPT 和课堂强调了什么。
3. 作业和 quiz 反复练了什么。
4. 哪些公式、定义、步骤最容易丢分。
5. 主观题应该怎么写才更像标准答案。

## 它适合谁

这个 Skill 适合：

* 正在期末周复习的大学生。
* 手里有 PPT、PDF、Word、Markdown、TXT、图片、表格、课堂转写、作业或往年卷的学生。
* 想让 AI 基于自己的资料整理复习内容，而不是让 AI 凭空发挥的人。
* 想生成考点清单、公式表、题库、模拟卷、Anki 卡片、错题本和复习计划的人。
* 想用 Codex / Claude Code / 类似 Agent 管理一门课复习材料的人。

它不适合：

* 完全没有资料，只想凭空生成一整门课题库。
* 需要保证 100% 正确答案的正式考试系统。
* 想替代自己核对原文和理解知识点的人。

这个项目会尽量让 AI 做“整理和提炼”的重活，但生成内容仍然需要你结合原始资料复核。

## 它和普通提示词有什么区别

普通提示词通常只告诉 AI：“帮我总结一下资料”“帮我出题”。

FinalReview-Skill 会更具体地约束 Agent：

* 先把资料转成 Markdown，再清洗，再提取，再出题。
* 按题源优先级判断重点，而不是平均总结。
* 真题、老师 PPT、作业、复习范围的权重不同。
* 速成课材料可以用来搭框架，但不能压过真题和老师资料。
* 每个知识点按重要程度生成 `1~6` 题，不机械平均。
* Markdown 输出默认题答分离。
* HTML/交互输出答案默认折叠。
* 客观题解析要写“这题核心知识点”。
* 主观题解析要写“这题答题核心点”“必须出现”“常见失分点”。
* AI 自己补的题必须标注 `AI补充` 或 `AI生成题`。

这让 Agent 的行为更稳定，也更贴近期末复习的真实需求。

## 可以处理哪些资料

只要你的 Agent 环境能读取或转换，FinalReview-Skill 都可以指导它处理：

* PPT / PPTX：老师课件、课堂 slides
* PDF：教材节选、复习范围、往年卷、扫描件
* Word：作业、实验报告、复习提纲
* Markdown / TXT：课堂笔记、转写文本、AI 初稿
* 图片：手写笔记、截图、扫描页
* Excel / CSV：实验数据、题目表、成绩规则
* 课堂录音转写：尤其适合提取老师强调过的内容

如果文件是扫描件或图片，Agent 应该尽量 OCR；如果 OCR 不可用或效果不好，Skill 会要求 Agent 明确标注，而不是假装已经读懂。

## 最终可以生成什么

一次完整的复习整理，通常可以生成：

```text
00_期末总复习指南.md
01_考点清单.md
02_公式表.md
03_练习题题库.md
04_高频题型.md
05_模拟卷A.md
06_模拟卷A_答案解析.md
07_错题本模板.md
08_Anki卡片.csv
09_知识图谱.md
10_复习计划.md
11_资料完整性检查.md
```

你不一定每次都要全套生成。比如你可以只让它做：

* “帮我提炼最高频考点”
* “只生成 3 天冲刺计划”
* “根据往年卷总结题型”
* “把这份 PPT 变成 Anki 卡片”
* “帮我出一套 90 分钟模拟卷”
* “检查我现在资料还缺什么”

## 项目结构

```text
FinalReview-Skill/
├── SKILL.md
├── AGENT.md
├── README.md
├── LICENSE
├── agents/
│   └── openai.yaml
└── references/
    ├── extraction-rules.md
    └── output-spec.md
```

每个文件的作用：

* `SKILL.md`：Codex Skill 的主入口。里面写了触发条件和核心工作流。
* `AGENT.md`：给 Claude Code、Codex 或其他 Agent 的长期规则文件。适合复制到项目级规则里。
* `README.md`：你现在正在看的项目说明。
* `agents/openai.yaml`：Codex UI 中显示 Skill 名称、简介和默认提示词。
* `references/extraction-rules.md`：详细的考点、公式、题型、完整性检查规则。
* `references/output-spec.md`：完整复习资料包的输出格式规范。

## 在 Codex 中安装

把这个仓库克隆或复制到 Codex 的 skills 目录。

Windows 默认路径通常是：

```text
C:\Users\<你的用户名>\.codex\skills\finalreview-skill
```

例如：

```powershell
cd C:\Users\<你的用户名>\.codex\skills
git clone https://github.com/Raymond-Specter/FinalReview-Skill.git finalreview-skill
```

如果你已经把仓库下载到了别的地方，也可以直接复制整个文件夹到：

```text
C:\Users\<你的用户名>\.codex\skills\finalreview-skill
```

安装后，重新打开 Codex 或开启一个新对话，让 Codex 重新加载 skills。

## 在 Codex 中使用

上传课程资料后，直接说：

```text
请使用 finalreview-skill，基于我上传的课程资料生成期末复习资料。
```

更具体一点：

```text
请使用 finalreview-skill，先读取这些 PPT、作业和往年卷，帮我整理：
1. 最高优先级考点
2. 必背公式
3. 高频题型
4. 一套模拟卷
5. 答案解析
6. 3 天复习计划
```

如果你快考试了，可以这样说：

```text
请使用 finalreview-skill。我还有 1 天考试，只生成冲刺版：
最高频考点、必背公式、最可能考的题型、20 道最值得做的题、答案解析。
```

如果你想让它更像老师带你复盘：

```text
请使用 finalreview-skill，基于这些资料帮我判断哪些内容最可能考。
每个考点请说明：
为什么重要、可能怎么考、我考试时应该怎么写、常见失分点是什么。
```

## 在 Claude Code 或其他 Agent 中使用

如果你的工具不支持 Codex 的 `SKILL.md` 自动发现，可以使用 `AGENT.md`。

做法很简单：

1. 打开本仓库的 `AGENT.md`。
2. 把内容复制到你的项目级 `AGENT.md`、`CLAUDE.md` 或其他规则文件中。
3. 在对话中告诉 Agent：

```text
请按 AGENT.md 里的期末复习规则处理这些课程资料。
```

这样即使不是 Codex，也能复用同一套复习协议。

## 推荐工作流

### 1. 准备资料

尽量把一门课相关资料放在一个文件夹里：

```text
course_materials/
├── 第1章_绪论.pptx
├── 第2章_Z变换.pdf
├── 作业题.docx
├── 往年卷.pdf
├── 复习范围.pdf
├── 课堂录音转写.txt
└── 手写笔记.jpg
```

资料越接近真实考试，效果越好。尤其推荐提供：

* 往年卷
* 老师 PPT
* 作业题
* 复习范围
* 课堂转写
* 答案或评分标准

### 2. 先让 Agent 做资料体检

可以先问：

```text
请使用 finalreview-skill，先不要出题。
请检查这些资料是否完整，告诉我还缺什么，以及哪些资料最值得优先看。
```

这一步能避免一上来生成一大堆看似完整、其实缺答案或缺真题的复习资料。

### 3. 再生成复习包

```text
请使用 finalreview-skill，基于这些资料生成完整复习包。
要求：
- 重点按真题、老师 PPT、作业、复习范围排序
- 题目和答案分开
- 所有 AI 补充题必须标注
- 答案解析要写常见失分点
```

### 4. 最后用问答查漏补缺

```text
请基于刚才整理出的资料回答：
最少拍控制器设计为什么要先确定 Φ(z)？
如果资料中没有足够依据，请直接说没有。
```

## 一些好用的提示词

### 生成考点清单

```text
请使用 finalreview-skill，从这些资料中提取考点清单。
按 high / medium / low 标注重要程度。
每个考点说明：可能考法、必须掌握的表述、来源。
```

### 生成公式表

```text
请使用 finalreview-skill，整理这门课的公式表。
每个公式请保留原式、变量含义、适用条件、常见错误和来源。
不要擅自改写公式。
```

### 根据往年卷总结题型

```text
请使用 finalreview-skill，优先分析往年卷。
总结高频题型、出现次数、对应考点、常见变式和复习建议。
```

### 生成模拟卷

```text
请使用 finalreview-skill，生成一套 120 分钟模拟卷。
题型尽量贴近往年卷。
题目和答案分开。
AI 补充题必须标注。
```

### 生成 Anki 卡片

```text
请使用 finalreview-skill，把这些资料整理成 Anki CSV。
字段为 front, back, tags, source。
优先生成概念卡、公式卡、易错点卡和简答题卡。
```

### 生成 1 天冲刺计划

```text
请使用 finalreview-skill，我明天考试。
请生成 1 天冲刺计划，只保留最可能提分的内容：
必背考点、必背公式、最值得做的题、最后 2 小时复盘清单。
```

## 题源优先级

FinalReview-Skill 很重视“题从哪里来”。默认优先级是：

1. 历年真题 / 往年卷
2. 老师 PPT / 课堂 slides
3. 平时作业 / 阅读作业 / quiz
4. 复习范围 / syllabus
5. 速成课题目
6. AI 补充题

为什么这样排？

因为期末复习不是知识百科竞赛。最该优先看的，是最可能出现在卷面上的东西。往年卷和老师 PPT 通常比泛泛的教材总结更接近考试。

速成课并不是没用。它适合帮你搭框架、看重点、抓易错点，但不应该压过你自己老师的材料。

## 答案解析风格

这个 Skill 要求解析服务于拿分，而不是只说“为什么对”。

例如一道主观题，普通 AI 可能会写：

```text
该方法的原理是……
```

FinalReview-Skill 更希望 Agent 写成：

```text
这题答题核心点：
1. 先写定义。
2. 再写关键公式。
3. 最后说明适用条件。

必须出现：
- 闭环脉冲传递函数
- 广义对象 G(z)
- 控制器 D(z)

常见失分点：
- 只写公式，不说明设计步骤
- 忘记检查可实现性
```

也就是说，它会尽量把“会不会”翻译成“考场上写什么”。

## 注意事项

FinalReview-Skill 会尽量保留来源，但它不是事实保证机器。

请务必注意：

* AI 可能读错扫描件或 OCR 文本。
* 老师临时划重点的信息可能没有出现在资料里。
* 没有答案的题，AI 的解析只能作为参考。
* 公式、定理和证明题一定要回到原始资料核对。
* 任何 `AI补充` 或 `AI生成题` 都不应被当作老师原题。

最好的用法是：让 Agent 帮你把资料变清楚，然后你用原文和课堂记忆做最后确认。
